Tentang Modul

Modul Program Loyalty digunakan untuk merencanakan, menjalankan, dan mengevaluasi program reward pelanggan — mulai dari reward poin, voucher belanja, hadiah, hingga benefit tenant. Tersedia dua jenis program: Standalone (independen, tidak terikat event) dan Event (terhubung ke event tertentu).

Langkah-Langkah Penggunaan
Dari membuat program hingga evaluasi bulanan
Hal. 2
1
Buat Program Baru

Klik + Tambah Program di pojok kanan atas. Isi nama, periode (tanggal mulai–selesai), target peserta, target member aktif, dan budget. Pilih tipe: Standalone atau Event.

💡 Program baru otomatis berstatus Aktif. Klik ikon toggle untuk menonaktifkan sementara.
2
Tambah Item Voucher atau Hadiah

Di dalam kartu program, klik + e-Voucher atau + Barang/Hadiah. Isi nilai nominal, stok awal, dan periode berlaku. Satu program bisa memiliki lebih dari satu jenis voucher/hadiah.

3
Input Realisasi Bulanan

Setiap bulan, klik + Input Realisasi pada item voucher atau hadiah. Isi jumlah yang terdistribusi/tersalurkan bulan ini. Data ini akan terakumulasi di Summary Bulanan.

⏰ Biasakan input setiap akhir bulan agar Summary Bulanan selalu akurat.
4
Pantau di Summary Bulanan

Buka menu Summary Bulanan di sidebar kiri. Gunakan filter bulan untuk berpindah periode. Halaman ini menampilkan KPI agregat, progress bar plan vs aktual, dan tren kumulatif dalam bentuk chart.

📌 Summary Bulanan — KPI seluruh program, progress bar, tren chart, dan form ACT analisa per program
5
Isi Analisa (ACT) di Summary Bulanan

Di bagian bawah halaman Summary, setiap program memiliki tiga kolom analisa yang harus diisi setiap bulan:

KolomIsi dengan
✅ HighlightPencapaian positif, momen berhasil, angka yang melampaui target
⚠️ KendalaHambatan yang ditemui, penyebab miss target
🔁 Tindak LanjutLangkah konkret untuk bulan depan, rencana perbaikan
💡 Tindak lanjut bulan ini akan otomatis muncul di bulan depan sebagai pengingat yang belum selesai.
6
Kunci Program (Lock)

Setelah semua realisasi dan analisa lengkap untuk periode ini, klik ikon gembok (🔒) di kartu program. Program yang terkunci tidak dapat diedit lagi kecuali dibuka oleh Admin.

⚠️ Lock bersifat permanen untuk pengguna biasa. Pastikan semua data sudah benar sebelum mengunci.

Tips & Referensi Penting
Hal-hal yang perlu diperhatikan saat menggunakan modul ini
Hal. 3
📅
Input realisasi rutin setiap bulan Summary Bulanan hanya akurat jika realisasi sudah diisi. Jadwalkan input di hari kerja pertama atau terakhir setiap bulan.
⚠️
Urutan yang benar: Program → Item → Realisasi Tidak bisa input realisasi jika belum ada item voucher atau hadiah di dalam program. Buat program dan item terlebih dahulu.
🔁
Kolom Tindak Lanjut berfungsi sebagai carry-over Isi kolom Tindak Lanjut dengan rencana konkret. Bulan depan, isian ini akan otomatis muncul kembali di kotak "Tindak Lanjut Bulan Lalu" sebagai pengingat.
🔒
Lock program hanya bisa dibuka oleh Admin Setelah terkunci, pengguna biasa tidak dapat mengubah data apapun di program tersebut. Hubungi Admin jika ada koreksi.

FAQ Umum
PertanyaanJawaban
Program saya tidak muncul di Summary? Pastikan program dalam status Aktif dan periode mencakup bulan yang dipilih.
Bisa tambah realisasi lebih dari satu kali sebulan? Ya. Sistem akumulasi — semua entri realisasi dalam satu bulan akan dijumlahkan.
Apakah data analisa bisa diedit setelah disimpan? Ya, selama program belum dikunci. Klik kolom analisa lagi dan simpan ulang.
Tren chart menampilkan data apa? Akumulasi kumulatif member, voucher, hadiah, dan benefit dari bulan Januari hingga bulan yang dipilih.
